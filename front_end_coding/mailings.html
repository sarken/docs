Rationalising our emails

Currently we have no style guide for the emails we send from the archive. These emails are important! The way we present them can change the way users experience the archive.

I suggest (as always) a rationalisation of mailings: a simple html template, minimal styling (colour branding), and a short style guide section to go in the Archive Style Guide I have begun developing:

Emails Style Guide

  • Mail notification views are in [view folder]
  • Mail notifications are written in HTML 4.01 and follow this template:
    [brief html template]
  • We do/do not include images. (decide)
  • When we include text that is normally parsed on the archive, we return the parsed/unparsed content. (decide)
  • We use simple friendly language, like this:
    [example]
  • When we reference an action, we always include the right link:
    Reply to comment

Note

Email styles have to be kept simple. There should be no positional layout whatsoever, as the only way to achieve this is with tables. I suggest using html 4 and only styling headings. Semantic HTML 4.01 should be employed. This is a small simple project, but requires input from ADT, Docs, and possibly Abuse/Admins.
